# 1. Introduction and Goals

### 1.1 Overview
`garden` is an Android application that helps users catalog plants and maintain them with AI-assisted identification and care recommendations. Users capture or upload photos, confirm plant identity, and track general and current care tasks.

### 1.2 Business Goals
- Let users register plants quickly with photo-first onboarding.
- Provide reliable AI-supported identification and health analysis.
- Improve plant-care consistency through actionable task guidance.
- Keep data local-first to reduce complexity in early product phases.

### 1.3 Quality Goals
1. **Usability:** Add and manage plants with minimal friction.
2. **Reliability:** Stable photo, storage, and task workflows.
3. **Privacy/Security:** No hardcoded API keys; protect user data and metadata.
4. **Maintainability:** Clear MVVM + Repository layering and test coverage.

### 1.4 Stakeholders
| Stakeholder | Interest |
| --- | --- |
| End users (hobby gardeners) | Easy cataloging, clear care guidance, trustworthy AI hints |
| Product/Producer | Deliver sprint goals, maintain roadmap focus |
| Android developers | Clean architecture, predictable conventions |
| QA | Reproducible flows, testable features, release confidence |
